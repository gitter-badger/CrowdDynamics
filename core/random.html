crowddynamics.core.random.random.poisson_clock[source]¶

Generated points in time using Poisson process.

Time between updates are independent random variables defined

\[\tau_i \sim \operatorname{Exp}(\lambda)\]

Where the rate parameter is

\[\operatorname{E}(\tau) = \frac{1}{\lambda} = interval\]

Times when agent updates its strategy

\[\begin{split}\begin{split}\begin{cases} T_n = \tau_1 + \ldots + \tau_n, & n \geq 1 \\ T_0 = 0 \end{cases}\end{split}\end{split}\]

Sequence of update times

\[\{T_1, T_2, T_3, \ldots, T_n\}, \quad T_n < \Delta t\]

Number of arrivals by the \(s\)

\[\begin{split}N(s) &= \max\{n : T_n \leq s\} \\ N(s) &\sim \operatorname{Poi}(\lambda)\end{split}\]
Parameters:
  • interval (float) – Expected frequency of update.
  • dt (float) – Discrete timestep \(\Delta t\) aka time window.
Yields:

float – Moments in the time window when the strategies should be updated.
