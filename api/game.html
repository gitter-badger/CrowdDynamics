Game¶

class crowddynamics.plugins.game.EgressGame(simulation, door, room, t_aset_0, interval=0.1, neighbor_radius=0.4, neighborhood_size=8)[source]¶

Patient and impatient pedestrians in a spatial game \((S, f)\) for egress congestion between players \(P \subset A\) with set of strategies \(S\) and payoff function \(f : S \times S \mapsto \mathbb{R}\).

Set of strategies

\[\begin{split}S &= \{ \text{Impatient}, \text{Patient} \} \\ &= \{ 0, 1 \}\end{split}\]

Payoff matrix / function

\[\begin{split}f(s_i, s_j) = \left[\begin{matrix}\left ( \frac{T_{aset}}{T_{ij}}, \quad \frac{T_{aset}}{T_{ij}}\right ) & \left ( -1, \quad 1\right )\\\left ( 1, \quad -1\right ) & \left ( 0, \quad 0\right )\end{matrix}\right]\end{split}\]

Estimated evacuation time for an agent

\[T_i = \frac{\lambda_i}{\beta},\]

where

  • \(\beta\) is the capacity of the exit door
  • \(\lambda_i\) number of other agents closer to the exit.

Average evacuation time

\[T_{ij} = \frac{(T_i + T_j)}{2}.\]

Effect on agents

  • \(k\)
  • \(\tau_{adj}\)
  • \(\sigma_{force}\)

crowddynamics.plugins.game.best_response_strategy[source]¶

New strategy is selected using best response dynamics which finds the strategy that minimizes loss using the payoff function

\[s_{i} = \underset{s \in S}{\arg \min} \sum_{j \in N_{i}^{neigh}} f(s, s_{j})\]

where

  • \(N_{i}^{neigh} \subset P \setminus \{P_{i}\}\) set is eight closes agents excluding the agent itself at maximum skin-to-skin distance of \(0.40 \ \mathrm{m}\) from agent \(i\).

crowddynamics.plugins.game.poisson_clock[source]¶

Poisson process that simulates when agents will change their strategies. Time between updates are independent random variables defined

\[\tau_i \sim \operatorname{Exp}(\lambda)\]

Where the rate parameter is

\[\operatorname{E}(\tau) = \frac{1}{\lambda} = interval\]

Times when agent updates its strategy

\[\begin{split}\begin{split}\begin{cases} T_n = \tau_1 + \ldots + \tau_n, & n \geq 1 \\ T_0 = 0 \end{cases}\end{split}\end{split}\]

Sequence of update times

\[\{T_1, T_2, T_3, \ldots, T_n\}, \quad T_n < \Delta t\]

Number of arrivals by the \(s\)

\[\begin{split}N(s) &= \max\{n : T_n \leq s\} \\ N(s) &\sim \operatorname{Poi}(\lambda)\end{split}\]
Parameters:
  • interval (float) – Expected frequency of update.
  • dt (float) – Discrete timestep \(\Delta t\) aka time window.
Yields:

float – Moments in the time window when the strategies should be updated.

crowddynamics.plugins.game.agent_closer_to_exit[source]¶

Agent closer to exit

Function mapping players to number of agents that are closer to the exit is denoted

\[\lambda : P \mapsto [0, | P | - 1].\]

Ordering is defined as the distances between the exit and an agent

\[d(\mathcal{E}_i, \mathbf{x}_{i})\]

where

  • \(\mathcal{E}_i\) is the exit the agent is trying to reach
  • \(\mathbf{x}_{i}\) is the center of the mass of an agent

For narrow bottlenecks we can approximate the distance

\[d(\mathcal{E}_i, \mathbf{x}_{i}) \approx \| \mathbf{c} - \mathbf{x}_{i} \|\]

where

  • \(\| \cdot \|\) is euclidean metric
  • \(\mathbf{c}\) is the center of the exit.

Algorithm

  1. Sort by distances to map number of closer agents to player

    \[\boldsymbol{\lambda}^{-1} = \underset{i \in P}{\operatorname{arg\,sort}} \left( d(\mathcal{E}_i, \mathbf{x}_{i}) \right)\]

  2. Sort by players to map player to number of closer agents

\[\boldsymbol{\lambda} = \operatorname{arg\,sort} (\boldsymbol{\lambda}^{-1})\]
Parameters:
  • points (numpy.ndarray) – Array [[x1, y2], [x2, y2]]
  • position (numpy.ndarray) – Positions of the agents.
Returns:

Array where indices denote agents and value how many agents are closer to the exit.
Return type:

numpy.ndarray
